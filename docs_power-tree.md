# Power Tree

## 1. Purpose

This document defines the full board power tree, including:
- input source
- switched rails
- generated rails
- consumer mapping
- design constraints

---

## 2. Power Overview

External supply:
- 12V DC input

Generated rails:
- `+12V_SW`
- `+5V_LED`
- `+3V3_LOGIC`
- `+10V_MOTOR`

Common reference:
- `GND`

---

## 3. Top-Level Power Flow

```text
12V DC Input
  │
  ├── DC Jack
  │
  ├── Power Switch
  │
  └── +12V_SW
       ├── MP2315 #1 → +5V_LED
       │               ├── WS2812B LEDs
       │               └── AMS1117-3.3 → +3V3_LOGIC
       │                                ├── Teensy 4.1
       │                                ├── TCA9548A
       │                                ├── OLED ×5
       │                                ├── HC4067 ×2
       │                                └── TB6612 logic VCC
       │
       └── MP2315 #2 → +10V_MOTOR
                       └── TB6612 VM
                           └── Fader motors ×5
```

---

## 4. Input Stage

## 4.1 DC Jack
- input voltage: 12V DC
- recommended connector:
  - DC-005-5A-2.5 or equivalent

### Pin assignment
- Pin1 = +12V input
- Pin2 = GND
- Pin3 = unused detect/switch terminal

---

## 4.2 Power Switch
Type:
- SPST rocker switch

### Wiring
- COM ← DC jack Pin1
- NO → `+12V_SW`

### Function
This switch controls the entire board power.

---

## 5. 12V Switched Rail

Rail name:
- `+12V_SW`

This is the main post-switch input rail.

### Consumers
- MP2315 #1 input
- MP2315 #2 input

### Constraints
- use wide copper
- route short from switch to power converters
- avoid sending this rail through thin logic routing

---

## 6. 5V Rail

Rail name:
- `+5V_LED`

Generated by:
- MP2315 #1

### Converter settings
- input: `+12V_SW`
- output target: about 4.8V to 5V
- feedback divider:
  - 100kΩ / 20kΩ

### Consumers
- WS2812B LED power
- AMS1117 input

### Important notes
1. This rail may carry significant current due to LED loading.
2. Worst-case WS2812B current can be about 1.5A for 25 LEDs at full white.
3. Use bulk capacitors along the LED rail.

---

## 7. 3.3V Rail

Rail name:
- `+3V3_LOGIC`

Generated by:
- AMS1117-3.3 from `+5V_LED`

### Consumers
- Teensy 4.1 logic
- TCA9548A
- OLED ×5
- HC4067 ×2
- TB6612 VCC
- optional logic pull-ups and low-power peripherals

### Important notes
1. This rail is for logic only.
2. Do not place motor or LED power loads on this rail.
3. Keep 3.3V routing clean and stable.

---

## 8. 10V Rail

Rail name:
- `+10V_MOTOR`

Generated by:
- MP2315 #2

### Converter settings
- input: `+12V_SW`
- target output: about 10.56V
- feedback divider:
  - 100kΩ / 8.2kΩ

### Consumers
- TB6612 VM pins
- through TB6612 to fader motors

### Important notes
1. This rail is motor-only.
2. Do not use it for logic or OLED modules.
3. Route with wide traces.
4. Place local bulk capacitance near TB6612 devices.

---

## 9. Ground Strategy

Rail:
- `GND`

All subsystems share common ground:
- input ground
- power converter ground
- logic ground
- motor driver ground
- LED ground
- ADC reference ground

### Important notes
1. Use a continuous ground plane if possible.
2. Keep high-current returns local.
3. Avoid forcing motor return current through sensitive analog sensing paths.

---

## 10. Power Consumer Summary

| Rail | Main source | Main consumers |
|------|-------------|----------------|
| +12V_SW | DC input + switch | MP2315 #1, MP2315 #2 |
| +5V_LED | MP2315 #1 | WS2812B, AMS1117 input |
| +3V3_LOGIC | AMS1117 | Teensy, TCA9548A, OLEDs, HC4067, TB6612 VCC |
| +10V_MOTOR | MP2315 #2 | TB6612 VM, fader motors |

---

## 11. Current and Load Considerations

### 11.1 Fader motor rail
Each fader motor:
- rated voltage: 10V
- max current: 800mA or less

Worst-case rough estimate:
- 5 × 800mA = 4A

Design implication:
- the 10V rail should be treated as a high-current rail
- inductor and routing must be sized accordingly

### 11.2 LED rail
WS2812B worst-case:
- about 60mA per LED at full white
- 25 × 60mA = 1.5A

Design implication:
- use strong 5V distribution
- use local bulk capacitors

---

## 12. Safety / Design Rules

1. Never connect the fader motor directly to 12V.
2. Do not power logic devices from the 10V rail.
3. Do not power OLEDs or TCA9548A from 5V if the logic domain is 3.3V.
4. Avoid thin traces on motor and LED supply rails.
5. Place decoupling close to all active devices.

---

## 13. Suggested Validation Tests

1. Verify no short between all power rails before power-up.
2. Verify `+12V_SW` after switch.
3. Verify MP2315 #1 output before connecting LEDs.
4. Verify MP2315 #2 output before connecting motors.
5. Verify `+3V3_LOGIC` before inserting logic modules.
6. Verify stable ground continuity across the board.
